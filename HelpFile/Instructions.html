Getting Started

with the

BEToolbox™

Initial Loading of Toolbox

 

  1. Back button.  This button acts exactly the same as the back button within internet browsers.  Pressing this button returns you to the previously viewed page or program.
  2. Print button.  This button pops up a dialog box which enables you to print a form.
  3. Options button.  Allows minimal configuration / functionality.
  4. Form browser.  This is the "browser" portion of the program.  This area is used by BEToolbox to display the various forms contained within the tool box: information pages, input pages, and output pages.  When visiting internet sites using the BEToolbox, the site will also be viewed in this portion of the form.
  5. Hide button. Hides the contents page.   This is useful for saving screen real-estate.  When pressed, the contents screen is hidden and the "Hide" button becomes a "Show" button.   When pressed again, the Contents page will be displayed.
  6. Book. Books contain two or more pages.   You open a book by double-clicking on the book's icon.
  7. Page.  A page contains data.   The data may be a help page or an individual program.  You view a page by single-clicking on the page icon.  
  8. (F & G):  Contents page.  This portion of the form is called the "Contents" page.  Located in the contents file are all the individual programs contained within BEToolbox and various help pages.

     

Program Information Form

 

  1. The selected program (selected by single-clicking the page icon).
  2. The expanded information form.
  3. The reduced input form.
  4. The reduced output form.

The program information page is essentially the help-file for the particular program.  The data on the information pages was taken from the latest available documentation for the VAX version of each program.  The documentation was modified as required during the conversion from VAX to Windows PC.    In many cases the documentation is extremely lacking as a result of very little original documentation being able to be found.

When a program is selected in the Contents form, the program is displayed as shown in the picture above.  The program consists of three parts: the Information page, the Input page, and the Output page.  When first loaded, the Information page is maximized while the Input and Output forms are reduced.  To enlarge a reduced form, place the mouse over the reduced form desired and single-click the mouse.  This will maximize the desired form while the currently active form will be moved to reduced mode.
 

Input Form

 

  1. The "Load File" button.  Used to load a previously saved input data into the form.
  2. The "Save File" button.  Used to save current input data to a file.
  3. "Other" buttons.  These may be buttons that display diagrams or help-codes, allow the user to add/delete problems, or select additional output files to be created.
  4. Reduced Information form.
  5. Reduced Output form.

The input form is used to input data into the program.    Each program has a highly individualized form, format, and data required.    Some forms are quite dynamic in that fields appear, disappear, or change, according to the options that have been selected.  While the forms differ greatly we have taken great care to ensure that all forms are standardized in "look and feel" so that the user does not have to relearn each program.

While data is being input on this form, the user may wish to consult documentation.  To do so, the user only needs to select the Information form.  The Input form will be reduced, but no data will be lost.  After consulting documentation, the user can return to the Input form by selecting it.

If data is being entered within one program and the user selects another page from the Contents page, all data in the current Input form will be lost.  Save your data to a file before doing this!

To obtain output from the data entered, single-click on the Output form.  This will reduce the Input form, maximize the Output form, process the data, and display the data on the Output form.  

 

Adding/Deleting Problems

 

In many cases, more than one "problem"s are needed to be input.  In these cases, "Add" and "Delete" buttons have been placed on the Input form.  By pressing the "Add" button, an additional data row (problem) is added to the input form (see below).  New problems will appear immediately below the last available problem.

In the same way, problems may be deleted by pressing the "Delete" button.  The problem to be deleted will always be the problem (row of boxes) immediately above the "Add" and "Delete" buttons.
 

Output Form

 

  1. Reduced Information form.
  2. Reduced Input form.
  3. Name of saved datafile used to produce the output.  If no file was saved or loaded, this field will be blank.
  4. The output data. (usually preceeded by an "echo" of the input data).

In most cases the output data is identical to that produced by the VAX versions of these programs.  Where changes were made, they were made only to correct severe difficiencies and do not change the general shape/form of the output data.

Output data cannot be saved to a file, although it may be printed.  To print the data, press the "Print" button from the toolbar.    Output data cannot be saved because it is easily retrieved with a previously saved input file.  Input data may be saved from the Input form.  To retrieve previous output data, load the previously saved Input data from a file and click on the Output form.
 
 

Hiding the Contents Form

 

  1. Show button.  After the "Hide" button is pressed, it changes to the "Show" button.  When pressed it displays the Contents page.

At times, more screen space is needed for the form browser to display all input fields or to display the output.  When more screen space is neeeded, press the "Hide" button on the toolbar.  This will hide the contents form and change the "Hide" button to a "Show" button.    To access the contents page again, just press the "Show" button.
 

Printing

 

To print the output of a program, press the "Print" button on the toolbar.  This will display a pop-up menu (shown above).  Unfortunately, this dialog box does not have all of the options as the "normal" windows print dialog.  You cannot specify the printer, paper, the orientation, or other options.  You can only print to the default printer with default paper and with the default orientation (in most cases 8x11 and portrait).  To print, select the top-most radio box (shown above as selected) and press the "Ok" button.
